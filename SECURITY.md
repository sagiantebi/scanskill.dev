# Security

## Reporting a vulnerability

Please report security issues **privately** so we can address them responsibly before public disclosure.

- Prefer your host’s **private vulnerability reporting** (for example GitHub **Security** → **Report a vulnerability**) if it is enabled for this repository.
- Otherwise, open an issue titled `SECURITY:` and describe the impact without posting a working exploit, or use a contact channel listed by the maintainers for this project.

Include:

- A short description of the issue and affected component (API, worker stage, frontend, etc.).
- Steps to reproduce or proof-of-concept at a safe level of detail.
- Affected versions or commit, if known.

We aim to acknowledge reports within a few business days. Thank you for helping keep users safe.
